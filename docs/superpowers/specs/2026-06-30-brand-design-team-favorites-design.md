# 브랜드전략디자인팀의 즐겨찾기 — 설계 문서

작성일: 2026-06-30

## 1. 목적

브랜드전략디자인팀이 함께 보는 **공유 즐겨찾기(북마크) 보드**.
팀원은 주소만 알면 누구나 보고, **관리자(나)만** 추가·수정·삭제한다.
애플 스타일의 깔끔한 디자인을 지향한다.

참고 사이트: https://ai-semina-ap4u.onrender.com/ (카드 그리드 형태의 사이트 모음)

## 2. 핵심 요구사항

- **보기**: 팀원 누구나 URL로 접속해 봄 (로그인 불필요)
- **편집**: 관리자만 추가/수정/삭제 (비밀 관리자 주소 + 토큰)
- **분류**: 카테고리로 묶고, 카테고리는 관리 모드에서 추가/수정/삭제 가능
- **정렬**: 카드 순서·카테고리 순서를 드래그로 변경 (1차 포함)
- **동기화**: 서버에 저장 → 모든 팀원·모든 기기가 같은 목록
- **디자인**: 애플 스타일(넉넉한 여백, 둥근 모서리, 은은한 그림자, 시스템 폰트, 다크모드)

## 3. 기술 스택

- **프레임워크**: Next.js (화면 + API 한 몸체)
- **배포**: Vercel (콜드스타트 없음, 깃허브 연결 자동 배포, Hobby 무료 플랜)
- **DB**: Neon Postgres (무료, 서버리스)
- 전부 무료 티어로 시작 가능

## 4. 전체 구조

```
[팀원 브라우저] ──보기──> [Next.js 앱 (Vercel)] ──> [Neon Postgres]
[관리자(나)]  ──비밀주소+토큰──> 추가/수정/삭제/정렬
```

## 5. 화면 구성 (애플 스타일)

- **헤더**: 사이트 제목 "브랜드전략디자인팀의 즐겨찾기" + 카테고리 탭 + 검색창
- **본문**: 북마크 **카드 그리드**
  - 카드: favicon + 제목 + 한 줄 설명, 클릭 시 새 탭으로 열림
  - hover 시 살짝 떠오르는 애니메이션
- **카테고리 탭**: 전체 / (사용자 정의 카테고리들) — 눌러서 필터링
- **디자인 토큰**:
  - 시스템 폰트(-apple-system, SF Pro 느낌)
  - 카드 radius ~16px, 은은한 그림자
  - 무채색 배경 + 포인트 컬러 1개
  - **다크모드 지원**
  - 넉넉한 여백

## 6. 데이터 구조

```
categories
  - id (PK)
  - name
  - sort_order        # 카테고리 탭 순서

bookmarks
  - id (PK)
  - title
  - url
  - description       # 한 줄 설명 (선택)
  - favicon_url        # 자동 수집, 실패 시 기본 아이콘
  - category_id (FK -> categories.id)
  - sort_order        # 카테고리 내 카드 순서
  - created_at
```

- favicon: URL 입력 시 자동 수집 (Google favicon API 또는 메타 파싱). 실패 시 기본 아이콘.

## 7. 관리자 편집 흐름 (핵심)

- 일반 주소 `/` 는 **보기 전용** — 추가/삭제/편집 UI 없음
- 관리자는 **비밀 관리자 주소**로 접속: 예 `/manage?key=<SECRET_TOKEN>`
  - `SECRET_TOKEN`은 Vercel 환경변수에 저장, 관리자만 앎
  - 토큰이 맞아야 편집 UI 노출 + 서버 API가 편집 요청 허용
  - 한 번 인증되면 브라우저 쿠키에 기억 → 그 기기선 이후 비번 입력 없이 관리 모드
- 토큰 없는 추가/수정/삭제/정렬 API 요청은 **서버가 거부**
- 토큰 틀리면 일반 보기 화면으로 폴백 (관리 UI 노출 안 함)

## 8. 주요 동작

- **추가**: 관리 모드에서 "+" → URL·제목·설명·카테고리 입력 → 저장 (favicon 자동)
- **수정/삭제**: 카드 hover 시 편집/삭제 아이콘
- **카테고리 관리**: 관리 모드에서 카테고리 추가/이름변경/삭제
- **정렬**: 카드와 카테고리를 드래그로 순서 변경 → 서버에 sort_order 저장
- **검색**: 제목·설명·URL 기준 즉시 필터 (클라이언트)

## 9. 예외 처리

- 잘못된 URL 입력 검증
- favicon 수집 실패 → 기본 아이콘
- DB 연결 실패 → 안내 메시지
- 빈 카테고리 / 검색 결과 없음 상태 디자인
- 토큰 불일치 → 보기 화면으로 폴백 (관리 UI 숨김)

## 10. 테스트

- **권한**: 토큰 있을 때만 추가/수정/삭제/정렬 가능, 없으면 모두 거부 (가장 중요)
- 카테고리 필터·검색 동작
- 정렬 변경 후 새로고침해도 순서 유지
- favicon 수집 폴백

## 11. 범위 / 단계

**1차 (이번 구현)**
- 카드 그리드 보기, 카테고리 탭/필터, 검색
- 관리자 추가/수정/삭제
- 카테고리 추가/수정/삭제
- 드래그 정렬 (카드 + 카테고리)
- 다크모드, 애플 스타일 디자인
- Neon Postgres 저장, Vercel 배포

**이후 고려 (YAGNI, 지금 안 함)**
- 개인 계정/다중 사용자, 누가 추가했는지 기록
- 태그, 즐겨찾기 가져오기/내보내기
- 링크 메타데이터 자동 미리보기(썸네일)

## 12. 열린 항목

- 실제 카테고리 이름은 즐겨찾는 사이트를 모아본 뒤 확정 (관리 모드에서 자유롭게 편집)
